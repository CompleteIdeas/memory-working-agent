How AWM Remembers

A plain-language tour of the memory system behind the agent — what happens when it learns something, how it finds the right memory later, and why it was built the way it was. No background required.

Most AI tools have no memory. Every conversation starts from zero. AWM (Agent Working Memory) gives an AI agent something closer to a human memory: it decides what's worth keeping, strengthens what it uses, forgets what it doesn't, links related ideas together, and — crucially — knows when it simply doesn't know. This page walks through exactly how, using everyday analogies and one running example you can follow the whole way down.

But the real question is why this is different from the other ways an AI can "have memory," and where that difference actually pays off in everyday use. That's the thread running through this whole page — and §7 lays it out head-to-head with concrete, day-in-the-life scenarios.

The running example we'll follow:

You tell your assistant: "My main project is called Atlas, and its deadline just moved from July 1 to August 15 because the scope grew." Later you ask: "When is Atlas due?" — and weeks after that, "What was the old Atlas deadline, and why did it change?" We'll trace how AWM stores this, finds it, and keeps the history.

1 · The big idea: memory is not a filing cabinet

The simplest way to give an AI memory is to dump everything into a big pile and, when asked a question, hand back whatever looks vaguely similar. That's how a lot of "AI memory" works. It's a filing cabinet: it stores, and it retrieves-by-similarity.

AWM is built on a different belief: remembering is an act of judgment, not storage. A good personal assistant doesn't recite every email you've ever received when you ask a question — they recall the right small handful, and if they genuinely don't know, they say so instead of making something up.

Analogy A filing cabinet keeps every scrap of paper and, when you ask for something, dumps the whole drawer on your desk. A seasoned librarian remembers which books matter, knows which ones connect to each other, lets the dusty unused ones drift to storage, and — if you ask for a book that doesn't exist — tells you "we don't have that" instead of handing you a random one. AWM is the librarian.

The filing cabinet (naïve RAG)

Keep everything → on a question, return the top-N most similar chunks → paste them into the prompt. No sense of importance, no links between facts, no forgetting, no "I don't know."

The librarian (AWM)

Decide what's worth keeping → strengthen what gets used → link related facts → let unused detail fade → and return a precise few, or nothing at all. Precision over volume, by design.

AWM would rather return nothing than return facts you didn't ask for — because a wrong memory poisons the agent's answer worse than a missing one.

2 · The science it's built on: ACT-R, in plain English

AWM didn't invent how memory should behave from scratch. It borrows from ACT-R, a model of human memory that cognitive scientists have refined for over 30 years. The single most important idea AWM takes from it is the base-level activation of a memory — a fancy term for a simple, familiar feeling:

Analogy Think about how easily a fact comes to mind. Your own phone number? Instant — you've used it thousands of times. A hotel room number from a trip three years ago? Gone. Memories you use often stay sharp; memories you don't use fade. That "how easily does this come to mind right now" strength is what ACT-R calls activation, and AWM computes it for every stored fact.

The formula looks intimidating but says something intuitive:

activation = ln(n + 1)  −  d · ln( age / (n + 1) )
SymbolPlain meaningEffect
nHow many times this memory has been usedMore uses → higher, sharper. (Your phone number.)
ageHow long since it was created / last touchedOlder → weaker, unless it keeps getting used.
d (0.3–0.6)The "forgetting rate" dialAWM nudges this per query: a precise lookup forgets faster (recent matters); broad exploration forgets slower (old memories surface).
Why this design Using a real cognitive model — not an ad-hoc "recency score" — means the behavior is principled and predictable: frequently-used knowledge persists, one-off noise decays, and the system has a built-in, biologically-motivated reason for every memory's current strength. AWM layers more signals on top (below), but this is the spine.

Two more borrowed ideas you'll see throughout:

"Fire together, wire together"

When two memories get recalled together and that turns out useful, AWM strengthens the link between them (called a Hebbian edge). Next time, recalling one gently pulls up the other — like how "salt" makes you think "pepper."

Sleep consolidation

Human brains replay and reorganize the day's memories during sleep. AWM runs a "sleep cycle" too (§5): it clusters related facts, strengthens important links, prunes duplicates, and lets trivia fade.

3 · Writing a memory: the bouncer at the door

When the agent learns something — "My main project is called Atlas…" — AWM does not just save it. It runs a short pipeline first.

  1. Understand it — embed

    The text is turned into a list of numbers (an embedding) that captures its meaning, so the system can later judge how similar two ideas are even when the words differ.

    Analogy Like assigning every sentence a precise GPS coordinate in "meaning-space." Sentences about the same thing land near each other, even if worded differently ("due date" sits near "deadline").

  2. Is this worth remembering? — salience

    A bouncer scores how important the fact is, from several cues: Is it surprising? Is it a decision? Does it carry consequences? Is it new, or just a restatement of something already known? Novelty counts most.

    VerdictWhat happens
    activeImportant enough to keep front-of-mind and link up.
    stagingBorderline — held in a waiting room; promoted only if it proves relevant later, else it expires.
    discardNoise — barely kept, easily forgotten.
    Why this design If everything is "important," nothing is. The bouncer is what keeps the store from drowning in trivia — the difference between a memory and a log file.

  3. Have I heard this before? — novelty & reinforce

    AWM checks for an existing memory about the same thing. If you repeat a fact, it doesn't make a clumsy duplicate — it strengthens the one it already has (bumps its confidence and use-count, the n from the ACT-R formula).

    Analogy Hearing a name twice doesn't create two people in your head — it makes you more sure of the one person. Repetition = a stronger single memory, not clutter.

  4. Did something change? — supersede

    Our example says the deadline moved from July 1 to August 15. AWM writes the new fact and marks the old one superseded — so normal recall returns "August 15," but the history isn't erased. Ask specifically for the old value and the "why," and it can still surface it.

    Example "When is Atlas due?" → August 15 (the live answer). "What was the old deadline and why did it change?" → July 1, moved because scope grew. A filing cabinet that simply overwrote the value could never answer the second question.
    Why this design Decisions have history, and history is knowledge: it stops the agent from quietly reverting to a known-bad choice, and lets it reason about "we tried X, it didn't work, so we chose Y." A plain notes file goes silently stale; AWM tracks the change.

  5. File it and link it — store & connect

    The memory is saved, and AWM looks for related memories to link it to (the "Atlas" facts wire together). These links are what later let it follow a train of thought from one fact to a connected one.

4 · Recall: the funnel from thousands to a few

Now you ask: "When is Atlas due?" The hard part isn't storing — it's finding the right one or two memories among thousands without dragging along junk. AWM does this as a funnel: cast a wide net, then narrow with progressively smarter (and more expensive) filters.

Analogy Hiring for a job. You don't interview everyone in the city. A cheap keyword filter pulls a few hundred résumés; a scoring pass shortlists a few dozen; you chase down a couple of references; then a senior expert actually reads the finalists and ranks them. Cheap-and-wide first, expensive-and-precise last. Recall works the same way.
Everything in memory thousands of facts
Wide net — keyword + meaning search a few hundred candidates
Score each one ACT-R strength + relevance + links + confidence
Follow the associations pull in connected facts
Expert re-read (reranker) the precision cut
A precise few — or nothing with a confidence score

Here's what each layer actually does:

  1. Untangle the question — coreference

    If you said "it" or "that," AWM first swaps in what you meant ("it" → "Atlas"), so the search isn't chasing a pronoun.

  2. Cast a wide net — keyword + vector search

    Two searches run at once: a keyword match (great at exact terms and IDs) and a meaning match using the embeddings from §3 (great when the words differ — "due date" finds "deadline"). Their results are pooled. This is the only step that decides what's even eligible.

    Why both Keyword search nails exact tokens but misses paraphrases; meaning search handles paraphrases but can drift. Running both and merging gets the strengths of each. This step is also AWM's known limit: a fact that's neither lexically nor semantically close to your query can fall outside the net — and nothing downstream can rescue what was never caught. That's why how a memory is written matters so much.

  3. Score every candidate — composite scoring

    Each surviving candidate gets a blended score from several signals — not just "how similar."

    SignalPlain meaning
    Text / meaning matchHow well it matches the words and the intent.
    ACT-R activationThe §2 strength — used-often-and-recent beats stale.
    Link strengthIs it well-connected to other relevant memories?
    Confidence gateShaky, low-confidence memories are held back.
    Proven-useful bonusMemories you've confirmed helpful before get a nudge.
    Why gated, not just added The relevance signal gates the others: a memory that's strong and well-connected but irrelevant to this question doesn't get to ride its popularity to the top. Relevance is the admission ticket; the rest are tie-breakers.

  4. Follow the train of thought — association walk

    Starting from the best matches, AWM walks the links between memories a step or two out, pulling in connected facts the raw search might have under-ranked — including the special case of who said it / which entity it's about, so "what does Caroline think about the trip" can favor Caroline's notes.

    Analogy One memory reminds you of another. Recalling "Atlas deadline" naturally brings up "Atlas owner" and "Atlas codename" — the same way remembering a friend's face brings up their name.

  5. The expert re-reads a generous shortlist — cross-encoder reranker

    A second, more careful model reads each finalist together with your question and re-scores relevance directly. This is the precision step — the senior reviewer who actually reads the candidates. Everything before it (the scoring in the last step) is really just a cheap, wide pre-filter whose job is to hand this expert a generous shortlist — not to pick the winner itself.

    Why this balance — and a real tuning win The expert is accurate but slow, so it reads a shortlist, not the whole store. We recently found the shortlist was cut too short — the cheap pre-filter was quietly dropping good answers before the expert ever saw them. Widening the shortlist let the expert rescue them: recall went up across the board with no loss of precision — the expert simply isn't fooled by the extra also-rans, and it correctly stays silent when nothing fits. The lesson: let the precise judge see plenty; let the cheap filter only keep out obvious noise.

  6. The "I don't know" guard — abstention

    Before answering, AWM checks whether the three independent signals (keyword, meaning, expert) actually agree this question is in-domain. If they don't, it returns nothing rather than a shaky guess.

    Example Ask "When is Atlas due?" and the signals agree → it answers August 15. Ask "What's the capital of the moon?" → no real match, the signals disagree → AWM stays silent instead of inventing an answer.
    Why this is the whole point For an agent, a confidently wrong memory is worse than a blank — it derails the answer and the user trusts it. Abstention is what makes AWM safe to wire directly into an agent's reasoning.

  7. Final answer, with a confidence number — confidence

    The top few memories are returned, each with a confidence score, so the agent knows how much to lean on them.

5 · Sleep: how memories settle overnight

Like a brain, AWM periodically runs a consolidation cycle — a "sleep." Nothing is being asked of it; it's reorganizing what it already has so future recall is sharper and cheaper.

🧩 Replay & cluster

Groups related memories together (all the "Atlas" facts), like tidying scattered notes into folders.

💪 Strengthen

Reinforces the links between memories that genuinely belong together.

🌉 Bridge

Builds shortcuts between clusters that keep co-occurring, so multi-step recall gets easier.

🍂 Decay & forget

Lets unused, low-value memories fade and eventually archive — capacity for what matters.

🧹 Prune duplicates

Merges near-identical notes so one strong memory replaces several weak echoes.

⚖️ Rebalance

Keeps any single memory from becoming an over-connected "hub" that hijacks every search.

Why bother — the counter-intuitive part You'd think more memories = better recall. The opposite is often true: clutter and duplicates bury the relevant fact under similar-looking noise. Consolidation has been measured to improve retrieval by removing that noise. Tidying the library makes the right book easier to find.
A subtle one: memories fade their detail, not their address When a memory goes long unused, AWM gradually compresses its contents while keeping the "cue" that finds it (its topic, links, and meaning-coordinate). Like an old memory where you recall that something happened and roughly what it was about, even if the fine detail has blurred — and recalling it freshens it right back up. This is why writing memories down and forgetting about them is safe.

6 · The big design decisions — and why

Every notable choice in AWM trades something away. Here's the reasoning behind the ones that define it.

DecisionThe trade-off, and why it's worth it
Precision over recallIt will sometimes miss a loosely-related memory, in exchange for almost never surfacing a wrong one. For an agent feeding its own reasoning, a clean small answer beats a noisy complete one.
Built on ACT-R, not ad-hoc scores30 years of cognitive science gives principled, predictable forgetting and strengthening — rather than a pile of hand-tuned heuristics that nobody can reason about.
Abstention (the "I don't know")Costs a little recall (it occasionally stays silent when it could have guessed right), buys enormous safety — the agent won't act on a hallucinated memory.
Wide net → expert rerank funnelThe accurate model is slow, so it only judges a shortlist. This keeps recall fast (tens of milliseconds, locally) while still getting expert-level precision on the finalists.
Local & in-processRuns on your machine with no network round-trip and no data leaving it. Fast and private, at the cost of using small (not giant) models — which the funnel design makes good enough.
Track changes (supersede), don't overwriteSlightly more storage and bookkeeping, in exchange for never losing decision history — the thing a notes file silently destroys.
Shared across agentsOne memory store can be read and written by many agents/tools at once, so a fact learned by one is instantly recallable by another — a shared brain, not siloed notebooks.
The one-line philosophy AWM is the precise, fast, durable memory layer; the judgment — what to chain together, what to ignore, when to act — is the agent's job. It hands the model the right small slice and gets out of the way.

Capabilities are abstract. Here's what each one feels like on an ordinary day:

☀️ Monday, and every Monday after

The advantage: continuity. You don't re-explain who you are, what Atlas is, or who's on the team — every morning, forever. A no-memory tool forgets overnight; a prompt-stuffing setup makes you (or the app) keep re-pasting the same context and pay for it every time. AWM just knows, and pulls only the slice that's relevant to today's question.

🔁 The decision that changed

The advantage: it won't backslide. Atlas's deadline moved July 1 → August 15. A notes file or plain-search system that overwrote the value can answer "what is it now" but has destroyed the history — so nothing stops the agent from later re-suggesting July 1, or repeating a plan you already tried and rejected. AWM keeps "we changed this, and here's why," so the agent reasons forward instead of in circles.

🗂️ The crowded topic

The advantage: precision under clutter. After months of use you have fifty notes that mention "budget." Plain search dumps a dozen near-identical ones into the prompt and the model gets distracted; the more you've stored, the worse it gets. AWM consolidates the duplicates, ranks by genuine relevance, and returns the one or two that matter — or, if your question really isn't covered, says nothing rather than padding the answer with lookalikes.

🙊 The question it shouldn't answer

The advantage: it doesn't make things up. You ask about something that was never actually discussed. Every other mode will cheerfully hand the model the "closest" text it can find, and the agent confidently states something false. AWM's three-signal agreement check makes it abstain — the single most important property for trusting an agent to act on its own memory.

💸 The cost line

The advantage: leverage on a cheap model. Because AWM hands the model the right small slice instead of a giant context dump, a small, inexpensive model can punch far above its weight. The headline result behind this project: a small model with AWM matched a frontier model's task accuracy at roughly 1/40th the cost — and recall runs locally in tens of milliseconds, with nothing leaving the machine.

👥 The shared brain

The advantage: learned once, known everywhere. A fact one agent learns — or one you tell it in a chat window — is instantly recallable by another agent, another tool, even a different model, because they share the same live store. Separate notebooks can't do this; a shared memory turns a fleet of agents into a team that actually remembers together.

The honest caveat (because trust matters) AWM is tuned for precision, so on a benchmark that simply asks "did you retrieve every related snippet?" it scores modestly on purpose — surfacing everything loosely-related is exactly the noise it's built to avoid. Judge it instead on the thing that matters in practice: does the agent get the right answer, cheaply, without hallucinating, session after session. That's the axis where the differences above show up.

8 · "Why not just use a bigger model / more context / plain search?"

"Just put everything in the prompt."

Context windows are large but not infinite, and they reset every session — close the chat and it's all gone. Stuffing thousands of facts in is also slow, expensive, and actually lowers accuracy (the model gets distracted by irrelevant text). AWM keeps a durable store and injects only the relevant slice.

"Just use plain similarity search (RAG)."

That's the filing cabinet from §1. It has no sense of importance, no links between facts, no forgetting, no change-tracking, and no "I don't know." It also re-reads files from scratch every time. AWM adds exactly those missing judgments — and is one a system can combine with RAG, using AWM for decisions/insights and RAG for bulk documents.

"Just use a bigger, smarter model."

A bigger model still has no memory between sessions, and following a long thread is the model's job — finding the right facts to think about is memory's job. The headline result behind this whole effort: a small, cheap model with AWM matched a frontier model's task accuracy at a tiny fraction of the cost. Good memory is leverage on top of any model.
