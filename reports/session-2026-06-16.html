2 · The memory gauntlet — a controlled benchmark

One independent variable: the memory substrate dropped into a byte-identical agent harness. Per-task filesystem wipe + a separate persistent store, deterministic scoring, pass^k with confidence intervals, and three metric axes (accuracy, tokens, recall latency). Arms:

awm — cognitive layer rag — same embeddings, plain cosine notes — flat lexical file longctx — full-dump context off — no memory

Result 1 — plain recall: AWM ≈ RAG (the honest part)

Hardened memory suite, k=3, memory-dependent pass-rate. AWM's interval overlaps RAG's — on single-fact recall there is no significant edge, which is expected: AWM uses the same retrieval. The no-memory ablation only proves "memory helps," not "AWM beats a good system."

Result 2 — long-context is strictly dominated

longctx spent the most tokens (~17k/task) and scored the worst (0% on memory-dependent tasks) — a byte-budgeted full dump evicts the older facts by probe time. "Just stuff it in the window" is the worst of both worlds at scale.

Result 4 — cross-agent shared recall: AWM wins by construction

Agent A records decisions; a different agent B (fresh identity) is asked about facts only obtainable from A's memory:

awm100%

A categorical gap, not a noisy margin. A memory library/file is per-process and structurally can't share across agents; AWM's shared substrate (one store + workspace, distinct agent ids) lets B recall what A wrote. This is the result that isn't a tie.

Methodology note: the first run showed a spurious rag 33% — a no-memory model guessed a plausible default that matched a loose regex. Tightening every probe to an unguessable specific dropped it to its true 0%. Lesson recorded: cross-agent probes must require a value only obtainable from the other agent's memory.
